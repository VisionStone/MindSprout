# MindSprout 后续开发建议

> 基于 v1.0 完成后的观察与思考

---

## 1. 测试覆盖

**现状：** 项目目前没有自动化测试。

**建议：**
- 使用 **vitest**（已在 devDependencies 中）添加单元测试
- **Store 测试**：appStore、canvasStore、mindmapStore 的状态变更逻辑
- **Canvas 渲染测试**：离屏 canvas 测试节点绘制、命中检测、坐标转换
- **数据库 CRUD 测试**：使用内存 SQLite 测试所有 CRUD 操作
- **IPC 测试**：模拟 `window.electronAPI` 进行前端组件测试
- **AI Service 测试**：Mock Llumiverse driver，测试提示词构建和结果解析

**优先级：** ⭐⭐⭐ 高 — 核心功能稳定前必须补齐

---

## 2. 布局算法扩展

**现状：** 仅实现了 dagre 层级布局。

**建议：**
- **径向布局 (Radial)**：根节点在中心，子节点按角度均匀分布，适合展示发散思维
- **力导向布局 (Force-directed)**：使用 d3-force 实现物理模拟，节点间有引力和斥力，适合探索性浏览
- **自由布局**：允许用户完全手动定位，关闭自动布局
- **布局切换动画**：切换布局时添加平滑过渡动画

**文件位置：** `src/canvas/LayoutEngine.ts`

**优先级：** ⭐⭐ 中 — 增强产品竞争力

---

## 3. 性能优化

**现状：** Canvas 2D 渲染已做基础优化（视口裁剪、脏标记）。

**建议：**
- **R-tree / 四叉树空间索引**：大量节点时加速命中检测（当前是 O(n) 遍历）
- **离屏缓存 (Offscreen Canvas)**：节点内容不变时缓存为 ImageBitmap，避免每帧重绘文本
- **LOD (Level of Detail)**：缩小时节点简化为色块，减少绘制指令
- **脏矩形重绘**：仅重绘变化区域而非全屏 clearRect
- **虚拟滚动**：思维导图列表使用虚拟列表，避免大量 DOM 节点

**文件位置：** `src/canvas/CanvasEngine.ts`, `src/canvas/HitTester.ts`

**优先级：** ⭐⭐ 中 — 导图超过 100 节点时明显受益

---

## 4. 导出功能

**现状：** PRD 列为未来扩展，当前仅支持 JSON 数据导入/导出。

**建议：**
- **图片导出**：Canvas `toDataURL('image/png')` 直接生成高清 PNG
- **PDF 导出**：使用 `html2canvas` + `jsPDF` 将画布转为 PDF
- **Markdown 导出**：将思维导图结构转为 Markdown 大纲
- **Freemind/XMind 导入导出**：支持 `.mm` 和 `.xmind` 格式

**优先级：** ⭐⭐ 中 — 用户高频需求

---

## 5. 自动保存

**现状：** 当前需手动触发保存（或关闭时提示）。

**建议：**
- 添加 `auto_save_interval` 设置（默认 30 秒）
- 使用 `setInterval` 或防抖机制自动保存当前导图
- 保存时显示 subtle 提示（如状态栏闪烁"已自动保存"）
- 意外关闭时恢复未保存的更改（通过临时文件或 localStorage）

**文件位置：** `src/pages/EditorPage.tsx`, `src/stores/appStore.ts`

**优先级：** ⭐⭐⭐ 高 — 数据安全基础功能

---

## 6. 撤销 / 重做 (Undo/Redo)

**现状：** 无撤销功能，误删节点无法恢复。

**建议：**
- 实现 **命令模式 (Command Pattern)**
- 每个操作封装为 Command：AddNodeCommand、DeleteNodeCommand、MoveNodeCommand、EditNodeCommand
- 维护 undo/redo 栈（限制深度如 50 步）
- 快捷键：Ctrl+Z / Ctrl+Shift+Z
- 批量操作（如 AI 扩展生成多个节点）作为单个复合命令

**文件位置：** 新建 `src/commands/` 目录

**优先级：** ⭐⭐⭐ 高 — 编辑器必备功能

---

## 7. 快捷键体系完善

**现状：** 基础键盘导航已实现。

**建议：**
- **快捷键面板**：Settings 中添加可查看的快捷键参考表
- **可配置快捷键**：允许用户自定义键位绑定
- **新增快捷键**：
  - `Ctrl+S` — 保存
  - `Ctrl+N` — 新建导图
  - `Ctrl+Shift+N` — 新建子节点
  - `Ctrl+Shift+E` — 编辑选中节点
  - `F2` — 重命名选中节点
  - `Ctrl+A` — 全选
  - `Ctrl+Shift+A` — 选中所有同级节点
  - `Escape` — 取消选择 / 关闭弹窗

**优先级：** ⭐⭐ 中

---

## 8. 节点样式增强

**现状：** 节点颜色仅按层级固定分配。

**建议：**
- **自定义颜色**：允许用户为单个节点设置背景色/文字色
- **图标支持**：节点标题前显示 Emoji 或自定义图标
- **边框样式**：实线/虚线/无边框
- **节点形状**：圆角矩形、圆形、菱形等
- **预设主题包**：提供多套配色方案（商务、创意、学术等）

**文件位置：** `src/canvas/NodeRenderer.ts`, `src/styles/global.css`

**优先级：** ⭐ 低 — 锦上添花

---

## 9. 搜索功能完善

**现状：** 思维导图列表支持按标题/描述搜索。

**建议：**
- **画布内搜索**：`Ctrl+F` 打开搜索框，高亮匹配的节点，支持跳转
- **全文搜索**：搜索节点标题、内容、描述
- **正则支持**：高级搜索模式
- **搜索历史**：保存最近搜索

**优先级：** ⭐⭐ 中

---

## 10. 多语言支持

**现状：** 界面为中文，AI 提示词也为中文。

**建议：**
- 使用 **SolidJS 的 context + signal** 实现 i18n
- 提取所有 UI 文本到 `locales/zh-CN.json` 和 `locales/en-US.json`
- 根据系统语言自动切换，可手动覆盖

**优先级：** ⭐ 低 — 面向国内用户可暂缓
